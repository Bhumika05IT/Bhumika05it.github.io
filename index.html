  • The Part Ordering API is built using C#, .NET Core, and SQL, while the Dashboard is developed using C#, .NET Core, Blazor Server, SQL, and Azure DevOps. The company's repair warehouse processes 2,000 to 3,000 laptops and tablets daily. Previously, technicians had to order missing parts using a legacy COBOL application, which had several limitations. The application could accept input but did not provide real-time database updates or show the availability of ordered parts. After placing an order, the warehouse team manually checked the availability of each required part. If all parts were available, they would assemble the package; if any item was missing, the package was placed on a separate rack, causing delays. Improvements with the New System The new system enhances the procurement process, improves warehouse operations, and provides repair technicians with real-time order status updates. It makes the repair workflow more efficient, predictable, and organized, reducing processing time by 30%. Automation is achieved through a .NET Core background thread, which processes pending orders as soon as parts become available. Key Features Integration with Dell’s API:

    • Fetches details such as pricing and supplier information. Selects the correct parts based on stakeholder requirements. Sends a request to Dell’s API when parts are required for a motherboard repair. Notifies the warehouse procurement team to manually post orders to Dell’s API when necessary.
    Dashboard Functionalities:
    • Allows users to mark orders as processing, ready for delivery, delivered, or posted. Generates a pick ticket with a barcode for all orders with available parts. Displays bin numbers for easy part retrieval. Enables warehouse runners to scan and verify orders using employee badge IDs before final delivery. My Responsibilities Collaborated with stakeholders to gather requirements, prototype the dashboard, and iterate based on user feedback. Designed, developed, and tested the entire system. Implemented enhancements, user training documentation, and maintenance. Managed the entire project independently, ensuring smooth deployment and ongoing improvements. This system significantly improves efficiency by automating order fulfillment, enhancing inventory management, and streamlining the repair workflow.
